# WriteLoop 기술 스택

이 문서는 현재 `WriteLoop` 프로젝트에서 실제로 사용 중인 기술 스택을 한눈에 볼 수 있도록 정리한 문서입니다.

## 1. 전체 구조

WriteLoop는 크게 아래 3개 영역으로 구성되어 있습니다.

- 웹 프론트엔드: `apps/frontend`
- 모바일 앱: `apps/mobile`
- 백엔드 API 서버: `apps/backend`

운영 환경에서는 Docker Compose로 여러 서비스를 함께 띄우고, Nginx가 프록시 역할을 담당합니다.

## 2. 웹 프론트엔드

위치: `apps/frontend`

### 핵심 기술

- Next.js 15
- React 19
- TypeScript
- CSS Modules

### 주요 용도

- 웹 서비스 메인 화면
- 작문, 피드백, 기록 보기 UI
- 로그인/회원가입/개인정보처리방침 등 웹 페이지

### 관련 설정 파일

- `apps/frontend/package.json`
- `apps/frontend/next.config.ts`
- `apps/frontend/app/`

### 비고

- Next.js의 `app` 라우터 구조를 사용합니다.
- 스타일링은 전역 CSS와 CSS Modules를 함께 사용합니다.
- 웹 프론트엔드 쪽에는 Capacitor 설정도 포함되어 있어, 웹 기반 빌드를 모바일 앱 패키징에 재사용하는 흐름도 일부 존재합니다.

## 3. 모바일 앱

위치: `apps/mobile`

### 핵심 기술

- Expo
- React Native
- Expo Router
- TypeScript
- React Navigation

### 주요 라이브러리

- `expo-router`
- `expo-secure-store`
- `expo-splash-screen`
- `expo-web-browser`
- `@react-navigation/native`
- `@react-navigation/bottom-tabs`
- `react-native-reanimated`
- `react-native-gesture-handler`
- `react-native-safe-area-context`
- `react-native-screens`

### 주요 용도

- iOS / Android 네이티브 앱
- 모바일 전용 작문 플로우
- 모바일 피드백/기록/로그인 화면

### 관련 설정 파일

- `apps/mobile/package.json`
- `apps/mobile/app.json`
- `apps/mobile/eas.json`
- `apps/mobile/src/app/`

### 비고

- Expo 관리형 워크플로우를 사용합니다.
- Android는 네이티브 `android/` 폴더를 함께 관리하고 있습니다.
- iOS 배포 준비는 EAS Build / EAS Submit 기준으로 설정되어 있습니다.

## 4. 백엔드

위치: `apps/backend`

### 핵심 기술

- Java 21
- Spring Boot 3.4
- Spring Data JPA
- Spring Data Redis
- Spring Mail
- Spring Security Core / Crypto
- Lombok

### 데이터 저장소 / 연동

- MySQL
- Redis

### 주요 용도

- 인증 및 소셜 로그인
- 작문 초안 저장
- 피드백 생성 API
- 기록 조회 API
- 메일 발송

### 관련 설정 파일

- `apps/backend/build.gradle.kts`
- `apps/backend/src/main/resources/application.yml`

### 비고

- ORM은 JPA / Hibernate 기반입니다.
- 기본 DB 드라이버는 MySQL Connector/J를 사용합니다.
- Redis는 draft/세션/보조 상태 저장 용도로 사용합니다.

## 5. AI / LLM 관련 스택

백엔드에서 AI 피드백과 코치 기능을 담당합니다.

### 지원 제공자

- Google Gemini
- OpenAI

### 현재 설정 구조

`application.yml` 기준으로 피드백 제공자와 코치 제공자를 분리할 수 있습니다.

- `llm.feedback-provider`
- `llm.coach-provider`

### 대표 모델 설정 예시

- Gemini
  - `gemini-3-flash-preview`
  - `gemini-3.1-flash-lite-preview`
- OpenAI
  - `gpt-5-mini`

## 6. 인프라 / 운영

### 핵심 기술

- Docker
- Docker Compose
- Nginx
- Redis

### 구성 서비스

`docker-compose.yml` 기준:

- `writeloop-frontend`
- `writeloop-backend`
- `writeloop-nginx`
- `writeloop-redis`

### 역할

- Nginx: 외부 트래픽 진입점, 리버스 프록시
- Frontend: Next.js 웹 서버
- Backend: Spring Boot API 서버
- Redis: 캐시 및 draft/상태 저장

## 8. 기술 스택 요약

한 줄로 요약하면, 현재 WriteLoop는 아래 조합의 프로젝트입니다.

- 웹: `Next.js + React + TypeScript`
- 모바일: `Expo + React Native + TypeScript`
- 백엔드: `Spring Boot + Java 21 + JPA + Redis + MySQL`
- 인프라: `Docker Compose + Nginx + Redis`
- AI: `Gemini / OpenAI 연동`
